Binary Tree

What is it?

- A Binary Tree is a hierarchical data structure where each node has at most two children: a left child and a right child.

- A Binary Search Tree (BST) is a special type of Binary Tree where the left child contains values less than the parent node, and the right child contains values greater than the parent node. BSTs are commonly used for efficient searching and sorting.

Time and Space Complexity

- In a Binary Tree, the structure is not constrained, and the complexity depends on the depth of the tree.

- In a Binary Search Tree (BST), the structure ensures faster searching, inserting, and deleting, assuming the tree is balanced.

Operation Binary Tree Binary Search Tree
Search O(n) O(log n)
Insert O(n) O(log n)
Delete O(n) O(log n)

Properties

  • The topmost node is called the root.
  • Each node can have at most two children.
  • Binary Trees can be balanced or unbalanced.
  • A Binary Search Tree (BST) enforces ordering: left child < parent < right child.
  • Used in searching, sorting, and hierarchical relationships.

Advantages and Disadvantages

✅ Represents hierarchical data effectively (Binary Tree).

✅ Efficient searching in balanced Binary Search Trees (BST).

✅ Supports dynamic resizing.

❌ Can become unbalanced, reducing efficiency (BST).

❌ Traversal can be time-consuming in unbalanced trees.

❌ Complex implementation for balancing in Binary Search Trees.
