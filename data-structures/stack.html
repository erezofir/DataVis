Stack

What is it?

A Stack is a Last-In-First-Out (LIFO) data structure where the last element added is the first to be removed.

Time and Space Complexity

Operation Time Complexity
Push O(1)
Pop O(1)
Peek O(1)

Properties

  • Follows Last-In-First-Out (LIFO) principle.
  • Supports efficient O(1) operations for push and pop.
  • Can be implemented using arrays or linked lists.
  • Used for managing nested function calls and recursion.
  • Simple and memory-efficient structure.

Operations

  • Push: Add an element to the stack.
  • Pop: Remove the top element from the stack.
  • Peek: View the top element of the stack without removing it.

Visualization

Interactive Code Example

Click "Run" to see the output or "Copy" to copy the code:

    
    // Initialize a stack
    Stack stack = new Stack<>();
    
    // Push elements to stack
    stack.push(1);
    stack.push(2);
    stack.push(3);
    
    // Peek the top element
    System.out.println("Top Element: " + stack.peek());
    
    // Pop an element
    System.out.println("Popped Element: " + stack.pop());
    
    // Display remaining stack
    System.out.println("Stack: " + stack);
    
      

    

    


    
    

      
Advantages and Disadvantages

      

        

          
✅ Simple and efficient.

          
✅ Minimal memory overhead.

          
✅ Easy debugging and testing.

        

        

          
❌ Limited size (in bounded stacks).

          
❌ No random access to elements.

          
❌ Overflow and underflow issues.
